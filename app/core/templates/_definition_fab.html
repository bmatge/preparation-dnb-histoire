{# Bouton flottant « Outils » partagé — mini-dictionnaire. Partial réutilisable entre matières : chaque matière (français, HG-EMC…) crée son propre `_tools_fab.html` qui se contente de définir les variables ci-dessous et d'inclure ce fichier. La logique HTML + JS vit ici une seule fois, les spécificités (URL de l'endpoint, libellés, identifiants de tests) sont passées en variables Jinja. Les ids DOM sont préfixés par `fab_root_id` pour permettre aux tests de vérifier la présence / absence de la FAB matière par matière. Variables Jinja attendues : - fab_root_id : id CSS de la racine (ex. "fr-fab-root"). - fab_endpoint : URL POST qui renvoie le fragment HTML résultat (HTMX : `hx-post`). - fab_matiere_label : libellé affiché dans le header de la popin (ex. "français", "histoire-géo-EMC"). - fab_placeholder : exemple de mots/expressions affiché dans l'input (ex. "alexandrin, hypallage…"). Le form POSTe via HTMX vers `fab_endpoint` avec un champ `term`. Le serveur répond un fragment HTML (voir `_definition_result.html`) qui est injecté dans la zone résultat. Pas d'évaluation JS côté client (contrairement à la calculette maths), tout passe par Albert derrière la route matière. #}
